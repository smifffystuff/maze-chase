# 01 — Skeleton App

## Goal
Bootstrap the project so every subsequent feature has a clean, correctly configured foundation to build on. The output is a running Next.js dev server with a blank canvas page, correct tooling, and all folder scaffolding in place.

## Steps

### 1. Initialise the project
```bash
npx create-next-app@latest maze-chase \
  --typescript \
  --app \
  --no-src-dir \
  --import-alias "@/*"
```
Then move files so source lives under `src/` manually (or re-run with `--src-dir` if supported in the target version).

### 2. Install and configure Tailwind CSS v4
Tailwind v4 uses a CSS-first approach — no `tailwind.config.js`.

```bash
npm install tailwindcss @tailwindcss/vite
```

In `src/app/globals.css`:
```css
@import "tailwindcss";

@theme {
  --color-primary: #ffdd00;
  --color-bg: #000000;
  --color-wall: #1a1aff;
  --color-ghost-frightened: #0000ff;
  --font-display: "Press Start 2P", monospace;
}
```

### 3. Install and initialise shadcn/ui
```bash
npx shadcn@latest init
```
Accept the defaults. Components will be added individually as features need them (`npx shadcn@latest add <component>`).

### 4. Create the folder structure
```
src/
  app/
    layout.tsx          # Root layout — sets dark background, imports global CSS
    page.tsx            # Entry point — renders <GameShell />
  game/
    engine/             # (empty — filled in 02-mvp)
    render/             # (empty — filled in 02-mvp)
    input/              # (empty — filled in 02-mvp)
    data/               # (empty — filled in 02-mvp)
    entities/           # (empty — filled in 02-mvp)
  components/
    GameShell.tsx       # Mounts the canvas, wires hooks, renders HUD overlay
    RotateOverlay.tsx   # Landscape-mode warning (filled in 05-orientation)
  hooks/
    useGameLoop.ts      # Stub — returns null for now
    useOrientation.ts   # Stub — returns { isPortrait: false } for now
  lib/
    utils.ts            # shadcn cn() helper (generated by shadcn init)
```

### 5. Root layout
`src/app/layout.tsx` should:
- Set `<html>` background to black
- Import the Google Font "Press Start 2P" for arcade feel (via `next/font/google`)
- Apply `min-h-screen bg-black text-white` to `<body>`

### 6. Game canvas page
`src/app/page.tsx` renders `<GameShell />`.

`src/components/GameShell.tsx`:
- Renders a `<canvas>` element that fills the viewport (`w-full h-full`)
- Holds a `canvasRef` passed to `useGameLoop`
- Renders a `<RotateOverlay />` on top (hidden for now)

### 7. Verify
- `npm run dev` starts without errors
- Browser shows a black full-viewport page
- `npm run build` succeeds
- `npx tsc --noEmit` passes

## Acceptance Criteria
- [ ] Dev server runs on `localhost:3000`
- [ ] Page is full-viewport black canvas
- [ ] Tailwind v4 theme tokens resolve (test with a temporary coloured div)
- [ ] shadcn/ui `cn()` helper importable from `@/lib/utils`
- [ ] All `src/game/` subdirectories exist (even if empty)
- [ ] TypeScript strict mode enabled, zero type errors
